Une brève description en 3 ou 4 lignes de l'index, de ses principales caractéristiques et fonctionnalités

DESCRIPTION DE L'INDEX

Element Description
Nom Nom de l'index (nom complet et abréviation si nécessaire)

Exemple : DOAJ (Directory of Open Access Journals)
Site web URL de la page d'accueil de l'index (ou URL spécifique vers la page Web de l'index)
Propriétaire Nom de la société ou de l'institution propriétaire de l'indice
Type de propriétaire Type d'organisation propriétaire de l'index (organisation à but non lucratif, organisme de recherche public, partenariat à but non lucratif (NPP), entreprise privée)
Pays du propriétaire Pays où se trouve le propriétaire de l'index
Date de lancement Année de lancement de l'index (peut être différente de la page d'accueil de l'index)
Périmètre Périmètre spécifique de l'index (scientifique, géographique, type d'accès, etc.)
Accès pour les utilisateurs Conditions dans lesquelles les utilisateurs peuvent accéder au service (gratuit, abonnement, etc.)
Accès pour les founisseurs de données Conditions permettant aux fournisseurs de données d'enregistrer leurs données dans l'index (fermé, ouvert, inscription requise, etc.)
Nombres d'objets indexés Nombre de ressources indexées (le type d'éléments peut varier et inclure des articles, des revues, des publications, etc.)
Documentation générale URL de la documentation sur le service (par exemple FAQ, fonctionnalités, accessibilité, etc.)
Documentation technique URL de la documentation sur les processus de collecte de données (par exemple, soumission manuelle, via un modèle de métadonnées, via un processus d'agrégation, etc.)

SERVICE FOURNI

Element Description
Type de service Typologie de services créée par le projet CRAFT-OA
Type de contenu Tous les types de ressources indexées (articles, revues, ensembles de données, etc.)
Langue du contenu Langue des ressources indexées
Provenance géographique du contenu Pays des ressources indexées
Niveau d'indexation des publications Niveau de publication traité par l'index (Journaux et/ou Articles)
Full text Gestion du texte intégral (affiché, par URL ou absent)
Sources de l'index Sources à partir desquelles l'index collecte des données (revues elles-mêmes, autres index, bases de données, agrégateurs, etc.)
Données collectées Contenus et/ou métadonnées
Standards supportés Standards de métadonnées et de contenus acceptés ou générés par l'index. Exemples : Dublin Core, PDF, etc.
Bibliodiversité Une brève description de la bibliodiversité de l'index et des éventuelles restrictions en termes de langues, de domaine scientifique, de provenance géographique ou de frais d'inscription.
Services additionnels Une URL et une brève description des autres outils liés à l'index (API, autres agrégateurs, outils de visualisation de données, etc.)

PROCESSUS D'INCLUSION

Element Description
Options d'inclusion Procédure détaillée à suivre par la revue pour rejoindre l'index : inscription, formulaire en ligne, évaluation de la revue, etc.
Processus de collecte des données Procédure détaillée que l'index suivra pour collecter les métadonnées de la revue (collecte manuelle ou automatique)
Formulaire pour les fournisseurs de données URL directe du formulaire de demande d'adhésion à l'index (si existant)
Informations de contact e-mail ou formulaire de contact utilisé pour contacter l'index

CONDITIONS MINIMALES

Element Description
Critères éditoriaux Une liste des principales conditions éditoriales que la revue doit appliquer pour faire partie de l'index
Formats de données Une liste des formats acceptés pour les publications (par exemple, HTML, PDF).
Standards de métadonnées Liste des standards que les métadonnées doivent suivre (par exemple, Dublin Core…). Dans cette section, ces conditions sont obligatoires contrairement à la précédente section "Standards supportés"
Formats de métadonnées Une liste des formats acceptés pour les métadonnées des publications ou des revues (par exemple XML, JSON).
Métadonnées minimales Liste des champs de metadonnées obligatoire à renseiner

CRITÈRES ADDITIONNELS

Element Description
Spécifications éditoriales facultatives Spécifications éditoriales facultatives (celles-ci peuvent être vérifiées par l'index, mais ne sont pas bloquantes)
Métadonnées recommandées Exigences recommandées en matière de métadonnées
Métadonnées facultatives Exigences facultatives en matière de métadonnées
Autres spécifications Autres champs de spécifications techniques
Spécifications SEO/UX Liste des critères supplémentaires requis par l'index pour améliorer l'expérience utilisateur et l'optimisation des moteurs de recherche

SOURCES DES INFORMATIONS

Element Description
Wikipedia URL de la page Wikipédia sur l'index
Page d'accueil de l'index URL de la page d'accueil de l'index
Page d'actualité de l'index URL de la page d'actualités dde l'index (articles de blog, événements, agenda, etc.)
